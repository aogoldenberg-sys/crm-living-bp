## Стандарт кода — human-grade-code (применять ВСЕГДА)

Перед написанием, правкой или ревью любого кода применяй правила из скилла `human-grade-code`:

| ИИ-привычка | Человеческая замена |
|---|---|
| Обрабатывает все ошибки везде | Только на границах системы |
| Комментирует «что» | Только «почему» |
| Длинные имена-описания | Короткие точные имена домена |
| Пишет «на будущее» | YAGNI — ровно под задачу |
| Абстракция сразу | Абстракция после 3-го повтора |
| Try/catch вокруг всего | Падать громко там, где баг |
| 5 вариантов | Одно решение + почему |

**Чеклист перед сдачей кода:**
- [ ] Можно удалить 20% без потери?
- [ ] Повторный запуск безопасен?
- [ ] Ошибки пойманы только на границах?
- [ ] Имена читаются без комментариев?
- [ ] Указано что проверить руками?

## Архитектурные правила (нарушать нельзя)

- `apps → adapter → core → schemas` — направление зависимостей
- `core` — ноль импортов Firebase/React/Workers. Только schemas.
- Деньги — только integer копейки. Никогда float.
- Даты — ISO UTC строки. Конвертация в TZ только на фронте.
- Файл ≤ 200 строк. Функция ≤ 40.
- Schema-first: сначала Zod, потом код.
- Append-only события: не редактировать, только добавлять.

## Режим симбиоза с человеком

1. Перед кодом — одно предложение: что делаю и почему так.
2. Спорное место → `// РЕШЕНИЕ: взял X потому что Y. Скажи если не так.`
3. После кода — 1-3 строки: что проверить руками конкретными шагами.
4. Не переписывать чужое без запроса. Минимум изменений.
